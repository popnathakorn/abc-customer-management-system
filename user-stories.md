# User Stories — ABC-CMS (ABC Customer Management System)

---

## Customer Service Staff

**US-01**
> **As a** Customer Service Staff,
> **I want to** register a new customer's personal information, registered address, and document delivery address into the system,
> **so that** the company has complete and accurate customer records for service and communication purposes.

**US-02**
> **As a** Customer Service Staff,
> **I want to** enroll a customer into a service package (e.g. 5G Next Speed, 599 ฿/month) and record the subscription start and end dates,
> **so that** the customer's active service is tracked and managed correctly.

---

## Operations / Back-office Staff

**US-03**
> **As an** Operations Staff,
> **I want to** view a daily report of newly registered customers,
> **so that** I can monitor business growth and follow up with onboarding processes.

**US-04**
> **As an** Operations Staff,
> **I want to** view a daily service subscription history report for each customer,
> **so that** I can audit subscription changes and ensure accurate billing records.

---

## System / IT Administrator

**US-05**
> **As a** System Administrator,
> **I want to** consume a RESTful API with defined parameters to submit and store customer subscription data,
> **so that** external systems and applications can integrate seamlessly with the customer database.

---

## Database Administrator

**US-08**
> **As a** Database Administrator,
> **I want to** run a SQL query to retrieve a new customer registration report for each day,
> **so that** I can access raw registration data directly from the database for analysis and auditing.

**US-09**
> **As a** Database Administrator,
> **I want to** run a SQL query to retrieve the service subscription history for each customer per day,
> **so that** I can perform direct database analysis and verify subscription records for auditing purposes.

---

## System / IT Administrator

**US-07**
> **As a** System Administrator,
> **I want** customer data to be stored in a well-structured database,
> **so that** data is consistent, queryable, and maintainable across all services.

---

## Subscriber (Customer)

**US-06**
> **As a** Subscriber,
> **I want** my personal details, addresses, and chosen service package to be accurately recorded,
> **so that** I receive the correct service and all correspondence is sent to the right location.
